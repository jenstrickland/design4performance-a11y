Q: What challenges do you face coding buttons?

  • Creation & maintenance of images
  • Internationalization & localization
  • Enhancing “clickability” (interaction)
  • Maintaining separation of content, style & behaviour in the code
  • CSS3 offers you a solution!

Why style buttons?

  • Make your call-to-action tantalizing. The more attention it gets, the clearer it is to the viewer what they should do.
  • CSS states – active, focus, hover, visited – are useful & create CSS hooks + opportunity for interactivity.
  • BUT, just because you can, doesn’t mean you should.
  • Please, use good taste & judgement.

What is CSS?

  • CSS is Cascading Style Sheets.
  • CSS allows for separation of content & style, which creates information flexibility.
  • Ideally, HTML is completely useful with no style or behavior (JavaScript or CSS).

What is CSS3?

  • CSS3 is the latest version of CSS, still in development by the CSS WG & browsers.
  • CSS3 is in varying stages of readiness. Some CSS3 is stable enough to use today while other parts are not.
  • If you do use CSS3 (and/or HTML5) today, be sure to build in appropriate fallbacks for non-modern browsers.
  • Order your CSS properties properly, too. Vendor prefixes followed by the non-prefixed declaration. Once the property is supported, the final one will over-ride the old ones. 
    -moz-border-radius:1em; -o-border-radius:1em; -webkit-border-radius:1em; border-radius:1em;

What about using CSS3 today?

  • Many CSS3 features are ready to use today in modern browsers.
  • Others can be added to browsers that do not support the tech natively using polyfills (also called shims or shivs). A polyfill, or polyfiller, is a piece of code (or plugin) that provides the technology that you, the developer, expect the browser to provide natively — flattening the API landscape (some call it Regressive Enhancement).
